## 2025-05-14 - [Empty States and Keyboard Accessibility]
**Learning:** Adding an empty state to a dashboard significantly improves the onboarding experience for new users by providing immediate feedback and instructions. Keyboard support (e.g., "Enter" to submit) in forms is a low-effort, high-impact accessibility and UX win.
**Action:** Always check for empty states in dashboards and ensure all form inputs support intuitive keyboard interactions.

## 2025-05-15 - [Explicit Labels and State-Dependent Actions]
**Learning:** Replacing cryptic, icon-only actions (like "X" for delete) with explicit text labels improves discoverability and reduces cognitive load. Coupling these with dynamic disabled states (e.g., disabling 'Undo' when there's nothing to undo) provides immediate, intuitive feedback about possible interactions.
**Action:** Prioritize explicit labeling over icons for critical actions and use standard HTML 'disabled' attributes with visual cues (opacity/cursor) for state-dependent buttons.
